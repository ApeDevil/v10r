# AGENTS.md

Agent-specific testing guidance for Velociraptor.

## Container-First Commands

All commands run inside the Podman container. Never run on host.

| Task | Command |
|------|---------|
| Run all tests | `podman exec v10r bun run test` |
| Run specific file | `podman exec v10r bun run test src/lib/server/db/errors.test.ts` |
| Run with filter | `podman exec v10r bun vitest run -t "pattern"` |
| Generate test schema | `podman exec v10r bun run db:test-schema` |
| Install deps | Rebuild container (never `bun install` on host) |

## Test Patterns

### Co-location

Tests live next to the source file:

```
src/lib/server/db/errors.ts
src/lib/server/db/errors.test.ts
```

State machine tests using Svelte runes use `.svelte.test.ts` extension:

```
dock.state.svelte.ts
dock.state.svelte.test.ts
```

### PGlite for DB Tests

```ts
let testClient: PGlite;

vi.mock('$lib/server/db', async () => {
  const { createTestDb } = await import('$lib/server/test/db');
  const { db, client } = await createTestDb();
  testClient = client;
  return { db };
});

const { myFunction } = await import('./my-module');
const { db } = await import('$lib/server/db');
```

- PGlite runs in-process — no external DB needed
- Migrations auto-apply via `drizzle/` folder (generated by `db:test-schema`, gitignored)
- Always close client in `afterAll`
- Clean tables in `beforeEach`

### vi.mock for Orchestration

```ts
vi.mock('$lib/server/db/notifications/mutations', () => ({
  getOrCreateSettings: vi.fn(),
}));
```

### Test Factories

`src/lib/server/test/fixtures.ts` provides factories:

- `makeUser(overrides?)` — user record
- `makeNotification(overrides?)` — notification record
- `makeConversation(overrides?)` — AI conversation record
- `makeDelivery(overrides?)` — delivery outbox record

## Schema Change Workflow

1. Modify schema in `src/lib/server/db/schema/`
2. Run `podman exec v10r bun run db:push` to sync changes to Neon
3. Run `podman exec v10r bun run db:test-schema` to regenerate test migrations
4. Update test fixtures if schema changed
5. Run `podman exec v10r bun run test`

## Boundaries

- **Never** run commands on host — always via `podman exec v10r`
- **Never** hand-edit files in `drizzle/` — they are auto-generated and gitignored
- **Never** use bare `bun test` — use `bun run test` (respects vitest config)
- **Never** import `hooks.server.ts` in tests — it starts auth/scheduler
- **Never** import real `$lib/server/db` without mocking — it connects to production Neon
