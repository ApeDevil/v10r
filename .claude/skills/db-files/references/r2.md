## Setup

### Client Configuration

```typescript
// src/lib/server/r2.ts
import { S3Client } from '@aws-sdk/client-s3';
import {
  R2_ACCOUNT_ID,
  R2_ACCESS_KEY_ID,
  R2_SECRET_ACCESS_KEY,
  R2_BUCKET_NAME,
} from '$env/static/private';

export const r2Client = new S3Client({
  region: 'auto', // R2 always uses 'auto'
  endpoint: `https://${R2_ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: R2_ACCESS_KEY_ID,
    secretAccessKey: R2_SECRET_ACCESS_KEY,
  },
});

export const R2_BUCKET = R2_BUCKET_NAME;
```

### Environment Variables

```env
# .env
R2_ACCOUNT_ID=your-account-id
R2_ACCESS_KEY_ID=your-access-key
R2_SECRET_ACCESS_KEY=your-secret-key
R2_BUCKET_NAME=your-bucket-name

# Optional: Public bucket domain
R2_PUBLIC_URL=https://files.yourapp.com
```

### Getting Credentials

1. Go to Cloudflare Dashboard → R2
2. Create bucket (if needed)
3. Click "Manage R2 API Tokens"
4. Create token with:
   - Permission: Object Read & Write
   - Specify bucket(s) for least privilege

**Account ID:** Found in Cloudflare Dashboard URL or R2 overview page.

## S3 Compatibility

### Supported Operations

| Operation | Status |
|-----------|--------|
| `GetObject` | Full |
| `PutObject` | Full |
| `DeleteObject` | Full |
| `HeadObject` | Full |
| `CopyObject` | Full |
| `ListObjectsV2` | Full (use V2, not V1) |
| `CreateMultipartUpload` | Full |
| `UploadPart` | Full |
| `CompleteMultipartUpload` | Full |
| `AbortMultipartUpload` | Full |
| Presigned URLs | Full |

### NOT Supported

| Feature | Status |
|---------|--------|
| Object Tagging | Not supported |
| ACLs | Not supported |
| Bucket Policies | Not supported |
| Versioning | Not supported |
| Lifecycle Rules | Not supported |
| POST presigned URLs | Not supported |

**Note:** Use `PUT` presigned URLs for uploads (not `POST` multipart form).

### Region Configuration

R2 region is always `"auto"`:

```typescript
const client = new S3Client({
  region: 'auto', // Required for R2
  // ...
});
```

For SDK compatibility, `"us-east-1"` and empty string also work (alias to `"auto"`).

### Checksums

R2 supports multiple checksum algorithms:

```typescript
import { PutObjectCommand } from '@aws-sdk/client-s3';
import crypto from 'crypto';

const checksum = crypto.createHash('sha256').update(buffer).digest('base64');

await r2Client.send(new PutObjectCommand({
  Bucket: R2_BUCKET,
  Key: key,
  Body: buffer,
  ChecksumSHA256: checksum, // Also: ChecksumCRC32, ChecksumCRC32C, ChecksumSHA1
}));
```

## Free Tier

### Limits (2025)

| Resource | Free Tier |
|----------|-----------|
| Storage | 10 GB |
| Class A ops (writes) | 1 million/month |
| Class B ops (reads) | 10 million/month |
| Egress | **Unlimited** (zero fees!) |

**Key advantage:** No egress fees. Serving files to users is free.

### Paid Pricing

| Resource | Price |
|----------|-------|
| Storage | $0.015/GB/month |
| Class A ops | $4.50/million |
| Class B ops | $0.36/million |
| Egress | **Free** |

### Class A vs Class B

| Class A (Writes) | Class B (Reads) |
|------------------|-----------------|
| `PutObject` | `GetObject` |
| `DeleteObject` | `HeadObject` |
| `CopyObject` | `ListObjectsV2` |
| `CreateMultipartUpload` | |
| `UploadPart` | |
| `CompleteMultipartUpload` | |

## Public Buckets

### When to Use

| Use Case | Method |
|----------|--------|
| User uploads (private) | Presigned URLs |
| Public assets (logos, docs) | Public bucket |
| Authenticated downloads | SvelteKit endpoint + presigned |

### Setup

1. **Enable public access** in R2 dashboard
2. **Connect custom domain** (optional but recommended)
3. **Access via:** `https://{custom-domain}/{key}` or `https://pub-{hash}.r2.dev/{key}`

### Custom Domain Benefits

- Cloudflare CDN caching
- Custom cache rules
- Image optimization
- Analytics

### Caching Configuration

Set `Cache-Control` on upload:

```typescript
await r2Client.send(new PutObjectCommand({
  Bucket: R2_BUCKET,
  Key: key,
  Body: buffer,
  ContentType: mimeType,
  CacheControl: 'public, max-age=31536000, immutable', // 1 year
}));
```

**Cache strategies:**

| Content Type | Cache-Control |
|--------------|---------------|
| Immutable (hashed filenames) | `public, max-age=31536000, immutable` |
| Avatars (may change) | `public, max-age=86400` (1 day) |
| Dynamic content | `public, max-age=300` (5 min) |

### Presigned URLs with Public Buckets

**Important:** Presigned URLs only work with S3 API endpoint (`<ACCOUNT_ID>.r2.cloudflarestorage.com`), NOT custom domains.

```typescript
// For uploads - always use S3 endpoint
const uploadUrl = await getSignedUrl(r2Client, command, { expiresIn: 300 });
// Returns: https://account.r2.cloudflarestorage.com/bucket/key?signature...

// For downloads - can use public URL if bucket is public
const publicUrl = `${R2_PUBLIC_URL}/${key}`;
```

### Serverless Considerations

**Ephemeral filesystem:**
- `/tmp` has 10 MB limit
- Files don't persist between invocations
- Stream directly to R2, don't write to disk

**Function timeout:**
- Hobby: 10 seconds
- Pro: 15 seconds (configurable to 60s)
- Enterprise: up to 900 seconds

For large uploads, use direct browser → R2 with presigned URLs.

## Image Optimization

### Cloudflare Images (with R2)

When using R2 with a public bucket + custom domain, you get access to Cloudflare Image Resizing.

**URL format:**

```
https://files.yourapp.com/cdn-cgi/image/{options}/{path}
```

**Options:**

| Option | Example | Description |
|--------|---------|-------------|
| `width` | `width=200` | Resize width |
| `height` | `height=200` | Resize height |
| `fit` | `fit=cover` | scale-down, contain, cover, crop, pad |
| `quality` | `quality=75` | 1-100 |
| `format` | `format=auto` | auto, webp, avif, json |

**Example:**

```typescript
// Original
const url = 'https://files.yourapp.com/avatars/user123.jpg';

// Thumbnail (80x80, WebP)
const thumb = 'https://files.yourapp.com/cdn-cgi/image/width=80,height=80,fit=cover,format=auto/avatars/user123.jpg';
```

### Pricing

- **$0.50 per 1,000 unique transformations** (30-day sliding window)
- Same image + same transformation = 1 count
- Free transformation of R2-stored images on paid plans

### SvelteKit Image Helper

```typescript
// src/lib/utils/image.ts
export function optimizedUrl(
  path: string,
  options: {
    width?: number;
    height?: number;
    fit?: 'cover' | 'contain' | 'scale-down';
    quality?: number;
  } = {}
): string {
  const { width, height, fit = 'cover', quality = 80 } = options;

  const transforms = [
    width && `width=${width}`,
    height && `height=${height}`,
    `fit=${fit}`,
    `quality=${quality}`,
    'format=auto',
  ].filter(Boolean).join(',');

  return `${R2_PUBLIC_URL}/cdn-cgi/image/${transforms}/${path}`;
}

// Usage
const avatar = optimizedUrl('avatars/user123.jpg', { width: 80, height: 80 });
```

## Troubleshooting

### Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `InvalidAccessKeyId` | Wrong credentials | Verify R2_ACCESS_KEY_ID |
| `SignatureDoesNotMatch` | Wrong secret or Content-Type mismatch | Check R2_SECRET_ACCESS_KEY, ensure Content-Type matches presigned |
| `NoSuchBucket` | Bucket doesn't exist | Create bucket in R2 dashboard |
| `AccessDenied` | Token lacks permission | Check token scope in R2 settings |
| `EntityTooLarge` | File exceeds limit | Use multipart for files > 5GB |
| CORS error | Missing CORS config | Configure CORS in R2 dashboard |

### CORS Configuration

Set via R2 dashboard or API:

```json
[
  {
    "AllowedOrigins": ["https://yourapp.com"],
    "AllowedMethods": ["GET", "PUT", "HEAD"],
    "AllowedHeaders": ["Content-Type", "Content-MD5"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

**For development:**

```json
[
  {
    "AllowedOrigins": ["http://localhost:5173"],
    "AllowedMethods": ["GET", "PUT", "HEAD"],
    "AllowedHeaders": ["*"],
    "MaxAgeSeconds": 3600
  }
]
```

### Debug Uploads

```typescript
// Add logging to track issues
async function uploadWithLogging(key: string, buffer: Buffer) {
  console.log(`Uploading to: ${key}`);
  console.log(`Size: ${buffer.length} bytes`);

  try {
    const result = await r2Client.send(new PutObjectCommand({
      Bucket: R2_BUCKET,
      Key: key,
      Body: buffer,
    }));
    console.log(`Success: ETag=${result.ETag}`);
    return result;
  } catch (error) {
    console.error(`Upload failed:`, error);
    throw error;
  }
}
```

### Verify Bucket Access

```typescript
// Health check endpoint
import { ListObjectsV2Command } from '@aws-sdk/client-s3';

export const GET: RequestHandler = async () => {
  try {
    await r2Client.send(new ListObjectsV2Command({
      Bucket: R2_BUCKET,
      MaxKeys: 1,
    }));
    return json({ status: 'healthy' });
  } catch (error) {
    return json({ status: 'unhealthy', error: String(error) }, { status: 500 });
  }
};
```

## Migration from S3

R2 is S3-compatible, so migration is straightforward:

### 1. Update Endpoint

```typescript
// S3
const client = new S3Client({ region: 'us-east-1' });

// R2
const client = new S3Client({
  region: 'auto',
  endpoint: `https://${ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: { accessKeyId, secretAccessKey },
});
```

### 2. Update Credentials

Replace AWS credentials with R2 API token.

### 3. Data Migration

Use rclone for bulk migration:

```bash
rclone copy s3:source-bucket r2:dest-bucket --progress
```

Or Cloudflare's Super Slurper (in dashboard) for large migrations.

### 4. Remove Unsupported Features

- Remove object tagging
- Remove ACL configurations
- Remove lifecycle rules
- Remove versioning config
